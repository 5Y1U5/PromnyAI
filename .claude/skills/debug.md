# デバッグ

## 使い方
ユーザーが `/debug` と入力したとき、またはバグ報告を受けたとき、このスキルを実行する。

## 手順（Autonomous Bug Fixing 準拠）

### 1. 調査（質問せずにまず自分で調べる）
- エラーメッセージ・ログ・スタックトレースを確認する
- 関連するソースコードを読んで原因を特定する
- 最近の変更（`git log --oneline -10`）を確認し、関連するコミットを探す

### 2. 再現テスト作成
- バグを再現するテストを書く（テストが失敗することを確認）
- テストがバグの本質を正確に捉えていることを確認する

### 3. 修正
- 最小限の変更で修正する
- 再現テストが通ることを確認する
- 既存テストが壊れていないことを確認する: `{{TEST_COMMAND}}`

### 4. 報告
以下の形式でユーザーに報告する:

```
## バグ修正レポート

### 原因
（根本原因の説明）

### 修正内容
（何をどう変更したか）

### テスト
- 再現テスト: tests/xxx — PASS
- 既存テスト: 全件 PASS

### 影響範囲
（修正が影響する範囲の説明）
```

## 注意事項
- バグ修正は `fix/` ブランチで行う
- 修正と同時にリファクタリングしない
- CI テスト失敗も同じフローで自律的に修正する
