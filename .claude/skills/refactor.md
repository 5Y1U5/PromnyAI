# リファクタリング

## 使い方
ユーザーが `/refactor` と入力したとき、このスキルを実行する。
対象ファイルやスコープはユーザーの指示に従う。

## 手順（Safe Refactoring 準拠）

### 1. 事前準備
- `git status` で未コミットの変更がないか確認する
- 未コミットの変更がある場合、先にコミットするかユーザーに確認する
- 対象範囲のテストが存在するか確認する
- テストがなければ、リファクタリング前にテストを作成する

### 2. 変更計画
- リファクタリングの目的と範囲をユーザーに提示する
- 変更を1つの責務ごとに分割する（ファイル分割とロジック変更を同時にやらない）
- 各ステップでコミットする計画を立てる

### 3. 実行
- 1ステップずつ変更を実施する
- 各ステップ完了後にテストを実行して、壊れていないことを確認する
- 機能追加を混ぜない — リファクタリングのみに集中する

### 4. 完了確認
- テスト全件実行: `{{TEST_COMMAND}}`
- `git diff` で差分レビューを行い、意図しない変更がないか確認する
- 変更サマリーをユーザーに報告する

## 注意事項
- リファクタリング中に新しいバグを見つけた場合、別ブランチ（`fix/`）で対応する
- ブランチ名は `refactor/対象の説明` を使う
