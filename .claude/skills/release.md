# リリース

## 使い方
ユーザーが `/release` と入力したとき、このスキルを実行する。
feature ブランチから main へのマージフローを管理する。

## 手順

### 1. リリース前チェック
- `git status` で未コミットの変更がないか確認する
- 現在のブランチが feature ブランチであることを確認する（main でないこと）
- テスト全件実行: `{{TEST_COMMAND}}`
- ビルド確認: `{{BUILD_COMMAND}}`

### 2. リモートへプッシュ
- `git push -u origin <ブランチ名>` でプッシュする
- CI が通るか確認する（`gh run list` で最新のワークフロー状態を確認）

### 3. PR 作成
- `gh pr create` で PR を作成する
- PR タイトルは70文字以内で変更内容を要約する
- PR 本文に変更サマリーとテスト計画を記載する

### 4. マージ（ユーザー確認後）
- ユーザーにマージ実行の確認を取る
- `gh pr merge` でマージする（デフォルトはマージコミット）
- ローカルで `git checkout main && git pull` して同期する
- feature ブランチを削除する

## 注意事項
- main への直接コミットは禁止
- テストが通らない場合はリリースを中止する
- マージ前に必ずユーザーの確認を取る
